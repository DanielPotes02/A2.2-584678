A2.2 LDA y rboles de Decisin

En este trabajo, se analizar una base de datos histrica que contiene informacin sobre los pasajeros del Titanic, con el objetivo de predecir la probabilidad de supervivencia de cada uno de ellos utilizando un modelo de regresin logstica. Esta tcnica es adecuada, ya que la variable a predecir ("Survived") es binaria: 1 significa que el pasajero sobrevivi, y 0 que no lo hizo.

Variables utilizadas:

  • Survived: Variable objetivo (0 = No sobrevivi, 1 = Sobrevivi).
  • Pclass: Clase del pasajero (1 = Primera clase, 2 = Segunda clase, 3 = Tercera clase).
  • Age: Edad del pasajero.
  • SibSp: Nmero de hermanos o cnyuges a bordo.
  • Parch: Nmero de padres o hijos a bordo.
  • Fare: Tarifa pagada por el pasajero.
  • Cabin_known: Indica si se conoce el nmero de camarote (1 = S, 0 = No).
  • Sex_male: Gnero del pasajero (1 = Hombre, 0 = Mujer).
  • Embarked_Q: Indica si el pasajero embarc en el puerto Queenstown (1 = S, 0 = No).
  • Embarked_S: Indica si el pasajero embarc en el puerto Southampton (1 = S, 0 = No).
  • Con estas variables, el modelo buscar patrones que permitan identificar qu tipo de pasajeros tuvieron mayor probabilidad de sobrevivir al hundimiento.

Para evaluar el desempeo del modelo de regresin logstica, se realiz una separacin de los datos en un conjunto de entrenamiento (80%) y un conjunto de prueba (20%). Al igual, se necesita verificar que las proporciones al dividir los datos con el fin de asegurar que la divisin de clases (sobrevivientes y no sobrevivientes) se mantuviera constante tanto en el conjunto de entrenamiento como en el de prueba.

from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import cross_val_score
import numpy as np

# Definir un rango de valores de alpha
alphas = [0.0001, 0.001, 0.01, 0.1, 1, 10]
scores = []

for alpha in alphas:
    tree = DecisionTreeClassifier(ccp_alpha=alpha, random_state=42)
    score = cross_val_score(tree, X_train_selected, y_train, cv=5).mean()  # Cambi cv=len(y_train) por cv=5
    scores.append(score)

# Seleccionar el mejor 
best_alpha = alphas[scores.index(max(scores))]
print("Mejor alpha:", best_alpha)


# Entrenar el rbol con poda
tree = DecisionTreeClassifier(ccp_alpha=best_alpha, random_state=42)
tree.fit(X_train_selected, y_train)

# Graficar frontera de decisin

x_min, x_max = X_train_selected.iloc[:, 0].min() - 1, X_train_selected.iloc[:, 0].max() + 1

from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score

# Predicciones
y_pred_lda = lda.predict(X_test_selected)
y_pred_tree = tree.predict(X_test_selected)

# Funcin para imprimir mtricas
def print_metrics(y_true, y_pred, model_name):
    print(f"\nMtricas para {model_name}:")
    print("Accuracy:", accuracy_score(y_true, y_pred))
    print("Precision:", precision_score(y_true, y_pred))
    print("Recall:", recall_score(y_true, y_pred))
    print("F1-score:", f1_score(y_true, y_pred))

# Evaluar modelos
print_metrics(y_test, y_pred_lda, "LDA")
print_metrics(y_test, y_pred_tree, "rbol de decisin")

Ventajas de cada modelo

  1. Anlisis Discriminante Lineal (LDA)
  • Mayor precisin y exactitud: LDA es ms confiable cuando es crucial minimizar los falsos positivos.
  • Mayor exactitud: tiene una ligera ventaja clasificando correctamente ms observaciones.
  1. rbol de Decisin podado
  • Mayor recall: Mejor identificando observaciones positivas y reduciendo los falsos negativos.
  • Mayor F1-score: Muestra un ligero mejor balance entre precisin y sensibilidad
  • Interpretabilidad visual: Se puede visualizar cmo toma decisiones en cada nivel.

En este caso, como ambos modelos tienen un desempeo similar, la eleccin depender de si es ms importante minimizar falsos positivos (precision - LDA) o minimizar falsos negativos (recall - rbol de Decisin).
